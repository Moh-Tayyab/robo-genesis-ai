# Pedagogical Layers (L1-L5)

## L1: Manual Layer

**AI Involvement**: None
**Focus**: Explain concepts without AI assistance

### Characteristics
- Student performs all work manually
- No AI code generation or assistance
- Focus on understanding fundamentals
- Building mental models

### Content Requirements
- Clear explanations with analogies
- Step-by-step manual procedures
- Hands-on exercises without AI
- Reflection questions

### Example Topics
- What is a robot?
- Understanding sensors and actuators
- Basic programming concepts
- Manual URDF creation

---

## L2: Collaboration Layer

**AI Involvement**: Assisted
**Focus**: AI-generated extensions AFTER student understanding

### Characteristics
- Student demonstrates understanding first
- AI provides variations or extensions
- Collaborative problem-solving
- Building on L1 knowledge

### Content Requirements
- Verify understanding before AI use
- AI generates variations of student work
- Compare manual vs AI approaches
- Discuss AI limitations

### Example Topics
- AI-assisted code review
- Generating test cases with AI
- Extending manual URDF with AI suggestions
- Debugging with AI assistance

---

## L3: Intelligence Layer

**AI Involvement**: Templated
**Focus**: Reusable templates, skills, and agents

### Characteristics
- Using pre-built AI templates
- Creating reusable AI skills
- Pattern-based AI assistance
- Building AI-native workflows

### Content Requirements
- Introduce AI templates
- Teach skill creation
- Practice pattern application
- Build reusable components

### Example Topics
- ROS 2 node templates
- URDF generation skills
- Reusable launch file patterns
- AI-assisted documentation

---

## L4: Spec-Driven Layer

**AI Involvement**: Guided
**Focus**: Module integration, plan generation, validation

### Characteristics
- AI generates from specifications
- Integration of multiple components
- Specification-first development
- Validation and verification

### Content Requirements
- Write specifications first
- AI generates implementation
- Validate against spec
- Integration testing

### Example Topics
- Generating ROS 2 packages from specs
- Multi-robot coordination
- System integration
- Automated testing

---

## L5: Full Autonomy Layer

**AI Involvement**: Autonomous
**Focus**: End-to-end AI-driven workflows

### Characteristics
- Student orchestrates AI agents
- AI handles end-to-end tasks
- Minimal human intervention
- Capstone-level projects

### Content Requirements
- Agent orchestration
- Multi-agent workflows
- Quality assurance
- Real-world deployment

### Example Topics
- Autonomous robot development
- Full project generation
- Production deployment
- Maintenance and updates

---

## Layer Progression

```
L1 (Manual) → L2 (Collaboration) → L3 (Intelligence) → L4 (Spec-Driven) → L5 (Autonomy)
```

**Rules:**
1. Students should complete L1 before L2
2. L3 assumes L1-L2 understanding
3. L4 requires L3 template knowledge
4. L5 is for advanced students only
